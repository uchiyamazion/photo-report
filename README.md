# 写真報告書作成アプリ

現場で撮った写真を、既存の「写真報告書送付用原紙」フォーマット（表紙＋写真貼付原紙・6枠/シート）と
同じレイアウトのExcelファイルとして自動出力するアプリです。

fron-kanri / ac-inspection / ky-katsudou と同じ構成で作っています。

- **フロント**: `index.html`（スマホブラウザで動作。GitHub Pagesなどにそのまま置けます）
- **バックエンド**: `Code.gs`（Google Apps Script。マスタ管理・写真保存・Excel生成を担当）

---

## 1. GAS側のセットアップ

1. https://script.google.com/ で新規プロジェクトを作成
2. `Code.gs` の中身をすべて貼り付けて保存
3. 関数選択で `initSetup` を選び、**1回だけ実行**
   - 実行時にGoogleアカウントの権限確認が出るので許可してください
   - 以下が自動作成されます
     - `写真報告書アプリ_マスタデータ`（型式・製造番号・工事内容を保存するスプレッドシート）
     - `写真報告書アプリ_写真`（Driveフォルダ、現状未使用・将来の写真保管用に予約）
     - `写真報告書アプリ_出力`（生成したExcelの保存先フォルダ）
4. 右上「デプロイ」→「新しいデプロイ」→種類「ウェブアプリ」
   - 実行ユーザー：**自分**
   - アクセスできるユーザー：**全員**（社内共有のみで良い場合は「全員（Googleアカウントが必要）」でも可。ただしその場合フロント側のfetchにログインが必要になる点に注意）
5. 発行された **ウェブアプリURL** をコピー

## 2. フロント側のセットアップ

1. `index.html` を開き、以下の行を書き換える

   ```js
   const GAS_URL = 'https://script.google.com/macros/s/【ここにデプロイIDを貼り付け】/exec';
   ```

2. GitHub Pages（`uchiyamazion` の他アプリと同じリポジトリ運用）などにアップロードして公開

## 3. 使い方

1. 「マスタ管理」タブで型式・製造番号・工事内容を事前登録
2. 「新規作成」タブで物件名・作業内容・作業日を入力
3. 「＋ 写真を追加」で現場写真を撮影→自動でリストに追加される
   - 各写真ごとに撮影日時（自動入力・編集可）／撮影場所／型式／製造番号／工事内容（プルダウン）を設定
4. 「Excelを作成」を押すとGAS側で
   - 元の「表紙」シート相当を再現
   - 写真は6枚ごとに1シートへ自動分割し、各写真は元のマクロと同じ縮尺・中央寄せでセルに配置
   - Googleドライブの `写真報告書アプリ_出力` フォルダにxlsxとして保存
5. 完了後に表示される「Excel(.xlsx)をダウンロード」リンクから取得

## 補足・カスタマイズポイント

- **表紙の宛先会社（パナソニックHVAC＆CCシステムズ株式会社／北海道支店 など）**は、マスタ管理タブの
  「宛先会社」から登録し、新規作成画面のプルダウンで選択する方式にしています。会社名・支店/部署名を
  複数登録しておけば、報告書ごとに切り替えて使えます。
- 写真は1シート6枚（元のxlsmと同じ）を超えると自動的に2シート目・3シート目…に分割されます。
- 画像はアップロード前にスマホ側で長辺1600px・JPEG品質0.75にリサイズしてから送信しています
  （通信量とGASのペイロード上限対策）。必要に応じて `index.html` の `MAX_IMAGE_DIMENSION` / `JPEG_QUALITY` を調整してください。
- マスタデータ（型式・製造番号・工事内容）は `写真報告書アプリ_マスタデータ` スプレッドシートを直接開いて
  一括編集することも可能です。

## 既知の制約

- Apps Script Web Appの1リクエストあたりの上限（約50MB）の範囲内で写真を送信してください。
  目安として、リサイズ後の写真なら1回に20〜30枚程度までが安全です。多い場合は複数回に分けて送信してください。
- `insertImage` はGoogleスプレッドシート上に画像を配置したあとxlsxへ書き出す方式のため、
  出力までに数秒〜十数秒かかることがあります（写真枚数に比例）。
